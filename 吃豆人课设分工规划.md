---

## 一、项目概述

使用 C++ 和 SFML 图形库，实现经典吃豆人游戏。游戏包含玩家控制的吃豆人、多种行为模式的幽灵敌人、随机生成的地图、豆子收集、计分系统及多关卡设计。项目重点体现面向对象的封装、继承与多态特性。

## 二、整体架构设计

### 类继承体系

```
GameObject (抽象基类)
├── Character (角色抽象类，继承 GameObject)
│   ├── Pacman          — 玩家角色
│   └── Ghost           — 幽灵抽象类
│       ├── Blinky      — 红色幽灵（追踪型）
│       ├── Pinky       — 粉色幽灵（伏击型）
│       ├── Inky        — 蓝色幽灵（随机协作型）
│       └── Clyde       — 橙色幽灵（逃跑型）
├── Item (道具抽象类，继承 GameObject)
│   ├── Dot             — 普通豆子
│   ├── PowerPellet     — 能量豆
│   └── Fruit           — 水果道具
└── Wall                — 墙壁

GameEngine              — 游戏引擎（核心控制）
├── MapManager          — 地图管理
├── CollisionManager    — 碰撞检测
├── ScoreManager        — 计分系统
├── InputHandler        — 输入处理
└── UIManager           — 界面管理

GameState (抽象基类)
├── MenuState           — 主菜单状态
├── PlayingState        — 游戏进行状态
├── PausedState         — 暂停状态
└── GameOverState       — 游戏结束状态
```

### 多态体现

1. **GameObject::update()** — 各子类实现不同的更新逻辑
2. **GameObject::render()** — 各子类实现不同的渲染方式
3. **Ghost::chase()** — 每种幽灵实现不同的追逐策略（策略模式）
4. **GameState::handleInput() / update() / render()** — 不同游戏状态有不同行为
5. **Item::onCollect()** — 不同道具触发不同效果

## 三、功能模块划分

### 模块 A：核心引擎 + 玩家系统 + 地图系统

| 功能 | 说明 |
|------|------|
| GameEngine | 游戏主循环、状态管理、帧率控制 |
| GameObject 抽象基类 | 定义通用接口 update/render/getBounds |
| Character 抽象类 | 位置、速度、方向、动画帧管理 |
| Pacman 类 | 键盘输入控制、移动逻辑、吃豆判定、动画状态机 |
| MapManager | 从文件加载地图、生成墙壁/豆子/道具、关卡切换 |
| Wall 类 | 碰撞体积渲染 |
| Dot / PowerPellet / Fruit | 道具类实现，各自的 onCollect 多态行为 |
| InputHandler | 键盘事件封装、方向队列（提前输入） |
| 碰撞检测基础 | AABB 碰撞检测工具函数 |

### 模块 B：幽灵 AI + 界面系统 + 音效系统

| 功能 | 说明 |
|------|------|
| Ghost 抽象类 | 幽灵基类：移动逻辑、状态机（追逐/逃跑/散开/重生） |
| Blinky | 追踪型 AI：直接追踪玩家位置 |
| Pinky | 伏击型 AI：预测玩家前方4格位置 |
| Inky | 协作型 AI：基于 Blinky 位置和玩家位置计算目标 |
| Clyde | 随机型 AI：近距离追逐、远距离随机游走 |
| 恐慌模式 | 能量豆后幽灵变蓝、移速降低、可被吃 |
| GameState 体系 | 菜单/游戏中/暂停/结束 四种状态的切换与渲染 |
| UIManager | 分数显示、生命显示、关卡提示、游戏结束画面 |
| ScoreManager | 计分规则、最高分记录（文件读写） |
| 音效管理 | 背景音乐、吃豆音效、死亡音效（SFML Audio） |

## 六、接口约定

为保证两人并行开发顺利，提前约定以下接口：

### 共享数据结构

```cpp
// 方向枚举
enum class Direction { Up, Down, Left, Right, None };

// 二维向量
struct Vec2 {
    float x, y;
    Vec2 operator+(const Vec2& other) const;
    Vec2 operator*(float scalar) const;
};

// 网格坐标（地图用）
struct GridPos {
    int row, col;
};

// 游戏配置常量
namespace Config {
    constexpr int TILE_SIZE = 32;       // 每格像素
    constexpr int MAP_WIDTH = 28;       // 地图列数

    constexpr float GHOST_SPEED = 80.f;

```

### 关键接口定义

```cpp
// GameObject 抽象基类
class GameObject {
public:
    virtual ~GameObject() = default;
    virtual void update(float deltaTime) = 0;
    virtual void render(sf::RenderWindow& window) = 0;
    virtual sf::FloatRect getBounds() const = 0;
    bool isActive() const;
    void setActive(bool active);

protected:
    Vec2 m_position;
    bool m_active = true;
};

// Ghost 抽象基类（多态核心）
class Ghost : public Character {
public:
    enum class State { Chase, Scatter, Frightened, Eaten };

    void update(float deltaTime) override;
    void render(sf::RenderWindow& window) override;

    void setFrightened();   // 进入恐慌模式
    void eaten();           // 被吃掉

protected:
    virtual void chase(const Vec2& pacmanPos, const Vec2& blinkyPos) = 0;
    // Blinky 重写为直接追踪
    // Pinky 重写为伏击
    // Inky 重写为协作
    // Clyde 重写为随机

    State m_state = State::Scatter;
    GridPos m_scatterTarget;  // 散开目标角落
};

// GameState 抽象基类
class GameState {
public:
    virtual ~GameState() = default;
    virtual void handleInput(sf::Event& event) = 0;
    virtual void update(float deltaTime) = 0;
    virtual void render(sf::RenderWindow& window) = 0;
};
```

## 八、开发规范

1. **分支管理**：各自在 feature 分支开发，合并到 dev 分支前需对方 review
2. **命名规范**：类名 PascalCase，函数/变量 camelCase，常量 UPPER_SNAKE_CASE
3. **提交规范**：`feat: xxx` / `fix: xxx` / `refactor: xxx`
4. **头文件保护**：统一使用 `#pragma once`
5. **编码格式**：UTF-8，缩进4空格
6. **每日同步**：每天简短沟通进度，遇到接口变更及时通知对方
