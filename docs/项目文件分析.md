# 吃豆人项目 — 文件分析

## 项目总览

使用 C++ + SFML 开发的吃豆人游戏，采用状态机模式管理游戏流程。

**完成度：约 30%** — 基类体系已搭建，类型定义统一，核心接口已约定，大部分子类仍为空壳。

---

## 目录结构


## 各文件详细分析

### 核心框架 (`core/`)

| 文件 | 状态 | 职责 | 说明 |
|------|------|------|------|
| `Types.h` | **有实现** | 共享类型 | 定义 `Direction`（方向枚举）、`State`（幽灵状态枚举）、`Point`/`Point2`/`GridPos`（坐标结构体），所有模块共用 |
| `Config.h` | **有代码** | 聚合头文件 + 配置 | 集中 include 所有模块头文件，`Config` 命名空间包含 TILE_SIZE、MAP_WIDTH、MAP_HEIGHT、SPEED、LIVES、HIGHSCORE_PATH 等常量 |
| `GameEngine.h/.cpp` | 有框架 | 游戏主循环 | 已实现：800×800 窗口创建、60fps 帧率限制、主循环骨架（processEvents→update→render）、ESC 关闭窗口 |
| `GameObject.h/.cpp` | **有实现** | 核心基类 | 引用 Types.h，定义 `GameObject` 抽象基类（纯虚函数：update、render、getBounds），含 m_position、m_active |
| `InputHandler.h/.cpp` | 接口已声明 | 输入处理 | 已声明：`handleEvents()`、`getDirection()`、`isActionTriggered()`，内部持有 `Direction m_direction`，实现待补充 |

### 游戏角色 (`entities/`)

| 文件 | 状态 | 职责 | 说明 |

## 已完成的工作

1. **类型统一** — 创建 `Types.h`，消除 `Direction`、`State`、`Point` 的三处重复定义
2. **合并冲突修复** — `Ghost.cpp` 和 `Character.h` 的 git 合并冲突已解决
3. **继承关系修正** — `Character` 改为 `public` 继承 `GameObject`
4. **InputHandler 接口** — 声明了 `handleEvents()`、`getDirection()`、`isActionTriggered()`
5. **Item 基类接口** — 定义了 `ItemType` 枚举和 `onCollect()`/`getScore()` 纯虚函数
6. **ScoreManager 完整实现** — 计分 + 最高分文件持久化（save/highscore.txt）
7. **Config 命名空间** — 统一游戏配置常量，含 HIGHSCORE_PATH

## 当前问题

1. **`main.cpp` 未接入 GameEngine** — 仍是 SFML 测试代码，使用硬编码绝对路径加载图片
2. **`GameEngine` 未集成 InputHandler** — processEvents() 只处理窗口关闭，未委托给 InputHandler
3. **`Dot.h` 声明了 override 但 .cpp 实现待补充** — onCollect() 和 getScore() 需要实现
4. **Ghost 构造函数 `speed(speed)` 参数名冲突** — 参数名和成员变量同名，会导致赋值给自身

## 建议的开发优先级

1. **地图系统**（MapManager + Wall）— 没有地图其他东西都没法测试
2. **Pacman 移动**（Pacman + InputHandler）— 核心玩法
3. **豆子系统**（Item + Dot + PowerPellet）— 收集逻辑
4. **幽灵 AI**（Ghost + Blinky/Pinky/Inky/Clyde）— 游戏挑战性
5. **状态管理**（GameState + 各子状态）— 游戏流程
6. **UI 和音效**（UIManager + SoundManager）— 体验完善
