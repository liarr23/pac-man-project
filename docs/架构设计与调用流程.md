# 吃豆人项目 — 架构设计与调用流程

## 一、类从属关系

```
GameEngine (最上层，只管窗口和状态切换)
├── InputHandler
├── unique_ptr<GameState>  ← 当前激活的状态
│
└── GameState (多态基类)
    ├── MenuState
    ├── PlayingState (游戏核心，持有所有游戏对象)
    │   ├── Pacman*
    │   ├── vector<Ghost*>  [Blinky, Pinky, Inky, Clyde]
    │   ├── vector<Item*>   [Dot, PowerPellet, Fruit]
    │   ├── MapManager*
    │   ├── ScoreManager
    │   └── UIManager*
    ├── PausedState
    └── GameOverState
```

### 各层职责划分

| 层级 | 类 | 职责 | 不关心什么 |
|------|-----|------|-----------|
| 引擎层 | GameEngine | 窗口管理、主循环、状态切换 | 不知道 Pacman、Ghost 的存在 |
| 状态层 | GameState 子类 | 各阶段的游戏流程控制 | 不关心窗口怎么创建、帧率怎么控制 |
| 协调层 | PlayingState | 串联所有游戏对象，驱动碰撞检测和事件分发 | 不关心每个对象内部怎么移动、怎么渲染 |
| 实体层 | Pacman、Ghost、Item | 各自的行为逻辑（移动、AI、收集效果） | 不关心谁在调用自己、游戏处于什么状态 |
| 工具层 | ScoreManager、MapManager、UIManager | 独立功能模块 | 不关心游戏流程 |

```
GameObject (抽象基类)

│

    ├── PowerPellet     — 能量豆

│
├── MenuState           — 主菜单
├── PlayingState        — 游戏进行中
├── PausedState         — 暂停
└── GameOverState       — 游戏结束
```

## 三、主循环流程

GameEngine 的主循环每帧执行三步，每步都委托给当前 GameState：

```cpp
while (window.isOpen()) {
    processEvents();   // 第一步：处理事件
    update(dt);        // 第二步：更新逻辑
    render();          // 第三步：渲染画面
}
```

### 3.1 processEvents — 事件分发

```
GameEngine::processEvents()
│
├── 窗口关闭 / ESC 键 → window.close()
│
├── m_inputHandler.handleEvents(event)
│   记录玩家按下的方向键
│
└── m_currentState->handleInput(engine, event)
    转发给当前状态处理
```

### 3.2 update — 逻辑更新（以 PlayingState 为例）

```
GameEngine::update(dt)
│
└── m_currentState->update(engine, dt)
    │
    PlayingState::update(engine, dt)
    │
    ├── 1. 获取输入
    │   └── direction = m_inputHandler.getDirection()
    │
    ├── 2. 更新 Pacman
    │   ├── pacman->setDirection(direction)
    │   └── pacman->update(dt)  ← 移动、动画
    │
    ├── 3. 更新所有 Ghost
    │   └── ghost->update(dt)  ← AI 决策、移动、状态切换
    │
    ├── 4. 碰撞检测：Pacman vs Item
    │   ├── item->onCollect() → 返回 ItemType
    │   ├── scoreManager.addScore(item->getScore())
    │   └── if (type == PowerPellet)
    │       └── 遍历所有 ghost → ghost->setFrightened()
    │
    ├── 5. 碰撞检测：Pacman vs Ghost
    │   ├── Ghost 是 Frightened → ghost->eaten() + 加分
    │   └── Ghost 是正常状态 → pacman 扣命
    │       └── 命用完 → engine.changeState(make_unique<GameOverState>())
    │
    ├── 6. 检查胜利条件
    │   └── 所有豆子吃完 → 切换下一关 / 胜利
    │
    └── 7. 更新 UI
        └── uiManager->update(score, lives)
```

### 3.3 render — 画面渲染

```
GameEngine::render()
│
├── window.clear()
│
├── m_currentState->render(window)
│   │
│   PlayingState::render(window)
│   │
│   ├── mapManager->render(window)     地图墙壁
│   ├── 遍历 item->render(window)      豆子、能量豆、水果
│   ├── 遍历 ghost->render(window)     四个幽灵
│   ├── pacman->render(window)         吃豆人
│   └── uiManager->render(window)      分数、生命值 HUD
│
└── window.display()
```

## 四、状态切换流程

### 4.1 状态切换图

```
                  ┌──────────────┐
                  │   MenuState  │
                  └──────┬───────┘
                   Enter │
                         ▼
         ┌───────────────────────────────┐
         │         PlayingState          │
         └────┬──────────┬───────────┬───┘
    Escape    │   命没了   │  豆子全吃完 │
              ▼          ▼           ▼
     ┌────────────┐ ┌─────────────┐ ┌──────────┐
     │ PausedState│ │GameOverState│ │ 下一关 /  │
     └─────┬──────┘ └──────┬──────┘ │  胜利     │
       ESC │          Enter │        └──────────┘
           ▼               ▼
    PlayingState      PlayingState (重开)
```

### 4.2 切换代码示例

```cpp
// PlayingState 中按 Escape → 暂停
void PlayingState::handleInput(GameEngine& engine, sf::Event& event) {
    if (event.type == sf::Event::KeyPressed && event.key.code == sf::Keyboard::Escape) {
        engine.changeState(std::make_unique<PausedState>());
    }
}

// PausedState 中按 Escape → 继续
void PausedState::handleInput(GameEngine& engine, sf::Event& event) {
    if (event.type == sf::Event::KeyPressed && event.key.code == sf::Keyboard::Escape) {
        engine.changeState(std::make_unique<PlayingState>());
    }
}
```

### 4.3 changeState 内部机制

```cpp
void GameEngine::changeState(std::unique_ptr<GameState> newState) {
    m_currentState = std::move(newState);
    // unique_ptr 自动销毁旧状态，新状态接管
    // 下一帧主循环就会调用新状态的 handleInput/update/render
}
```

## 五、道具收集调用链

以吃能量豆为例，展示完整的消息传递：

```
PlayingState::update()
│
├── 碰撞检测：Pacman 碰到 PowerPellet
│
├── ItemType type = powerPellet->onCollect()
│   │
│   PowerPellet::onCollect()
│   ├── setActive(false)           // 自身从场景消失
│   └── return ItemType::PowerPellet  // 告诉调用方"我是能量豆"
│
├── scoreManager.addScore(powerPellet->getScore())
│   加 50 分
│
└── if (type == ItemType::PowerPellet)
    │
    ├── ghost1->setFrightened()
    ├── ghost2->setFrightened()
    ├── ghost3->setFrightened()
    └── ghost4->setFrightened()
        │
        Ghost::setFrightened()
        ├── state = State::Frightened
        └── scarytime = 0           // 开始计时
```

**关键设计：Item 不碰 Ghost，Ghost 不碰 Item，PlayingState 负责串联。**

## 六、输入传递链

```
玩家按下方向键 ↑
│
├── sf::Event: KeyPressed (Up)
│
├── GameEngine::processEvents()
│   ├── m_inputHandler.handleEvents(event)
│   │   └── m_direction = Direction::Up
│   │
│   └── m_currentState->handleInput(engine, event)
│       MenuState 不处理方向键，忽略
│
├── GameEngine::update(dt)
│   └── PlayingState::update(engine, dt)
│       ├── direction = m_inputHandler.getDirection()  → Direction::Up
│       ├── pacman->setDirection(Direction::Up)
│       └── pacman->update(dt)
│           └── 按 Up 方向移动，检查墙壁碰撞
│
└── GameEngine::render()
    └── pacman->render(window)  ← 面朝上方绘制
```

## 七、共享类型体系

所有共享类型统一定义在 `src/core/Types.h`，通过 `Config.h` 聚合引用：

```
Types.h
├── Direction { Up, Down, Left, Right, None }
├── State { Chase, Scatter, Frightened, Eaten }
├── Point { x, y, operator==, operator< }
├── Point2 { x, y, operator+, operator* }
└── GridPos { row, col }

Config.h
├── #include "Types.h"
├── #include 所有模块头文件
└── namespace Config
    ├── TILE_SIZE, MAP_WIDTH, MAP_HEIGHT
    ├── PACMAN_SPEED, GHOST_SPEED
    ├── INITIAL_LIVES
    └── HIGHSCORE_PATH
```
